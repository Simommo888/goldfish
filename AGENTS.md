# AGENTS.md

## 项目说明

这是一个 Obsidian / Markdown 个人知识库，用于长期沉淀个人知识、AI 应用开发经验、Agent 开发经验、Prompt、项目复盘、报错解决方案、商业想法和输出内容。

## 工作规则

1. 所有内容优先使用 Markdown。
2. 不要随意删除用户笔记。
3. 不要随意重命名已有文件。
4. 新增知识优先放入合适目录。
5. 不确定放哪里时，放入 `00_Inbox`。
6. 每篇重要笔记必须包含 YAML frontmatter。
7. 重要笔记要增加 Obsidian 双链引用。
8. 新增笔记时尽量使用 `07_Templates` 中的模板。
9. 修改前先理解目录结构和相关 MOC。
10. 不要安装依赖，除非用户明确要求。
11. 不要生成无意义空文件。
12. 完成后说明新增或修改了哪些文件以及如何使用。

## 文件命名规则

1. 文件名可以使用中文。
2. 项目类文件使用清晰项目名。
3. Prompt 文件名应包含使用场景。
4. 报错文件名应包含技术栈和问题关键词。
5. 永久笔记文件名应表达一个清晰观点。

## 笔记规范

每篇重要笔记建议包含：

- type
- status
- area
- tags
- created
- updated

## AI 使用说明

这个知识库未来会被 AI 读取，所以内容应尽量清晰、结构化、可检索、可复用。新增内容要优先服务学习、开发、求职和技术变现。

# AI 情报 Agent 维护规则

1. 修改 `goldfish` 前先阅读 `scripts/goldfish/README.md`。
2. 不要提交 API Key。
3. 新增来源必须写明 reason 或 notes。
4. 新增人物必须只追踪公开专业信息。
5. 不追踪私人生活和八卦。
6. 抓取失败时要优雅降级。
7. 日报内容必须可沉淀到知识库。
8. 重要修改后建议 GitHub 备份。
9. 小紫提醒：有复用价值的信息要沉淀，懂吗！
