# AI 信息源维护说明

## 官方源

官方源优先收录 OpenAI、Anthropic、Google DeepMind、Google Research、Hugging Face、Microsoft、Meta AI、NVIDIA 等公开新闻和博客。

## 论文源

论文源优先关注 arXiv cs.AI、cs.CL、cs.LG、Hugging Face Daily Papers、Papers with Code。论文是否值得入库，关键看能否转化成工程方法、评测方法、产品能力或长期观点。

## 开源项目源

开源项目源包括 GitHub Trending、Trendshift、Hugging Face Spaces。重点关注 Agent、RAG、LLM App、AI Coding、MCP、向量数据库、知识库和开发者工具。

## 产品源

产品源包括 Product Hunt AI、Product Hunt AI Agents、Hacker News AI 搜索入口。第一版以人工查看占位为主，不做复杂爬虫。

## 人物源

人物源只追踪公开专业信息：博客、Newsletter、公司新闻、GitHub、公开演讲、公开视频。不要追踪私人生活、八卦、粉丝争论或需要登录的内容。

## 添加新来源的标准

1. 与 AI 应用开发、Agent、RAG、AI Coding、MCP、开源模型、商业化机会相关。
2. 来源公开、稳定、可引用。
3. 有 RSS 优先填 RSS；没有 RSS 标记为“待人工查看”。
4. `notes` 必须说明为什么值得关注。

## 删除低质量来源的标准

1. 长期无更新。
2. 大量重复转载。
3. 标题党、无来源爆料、情绪化争论过多。
4. 需要登录或存在明显反爬限制。
5. 与个人重点方向长期无关。

## 判断信息是否值得入库的标准

1. 是否能形成长期观点。
2. 是否能提升 AI 应用开发效率。
3. 是否能启发 Agent / RAG / AI Coding 项目。
4. 是否能变成 Prompt、模板或工程经验。
5. 是否能带来商业化机会。
6. 是否有公开来源和可验证链接。
